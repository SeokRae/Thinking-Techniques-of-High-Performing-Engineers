# 챕터 개선 체크리스트

**프로젝트**: 챕터 마크다운 문서 품질 개선
**작성일**: 2025-11-29
**버전**: v1.0
**참조**: PRD-chapter-improvement.md

---

## 📋 전체 프로세스 개요

각 챕터 개선은 **7단계 프로세스**를 따릅니다:

```
Phase 1: 사전 준비
   ↓
Phase 2: PDF 매핑
   ↓
Phase 3: 내용 분석
   ↓
Phase 4: 구조 설계
   ↓
Phase 5: 내용 작성
   ↓
Phase 6: 품질 검증
   ↓
Phase 7: 최종 검토
```

---

## ✅ Phase 1: 사전 준비 (각 챕터별)

### 목표
현재 챕터의 상태를 파악하고 개선 방향 설정

### 체크리스트
```markdown
□ 챕터의 현재 마크다운 파일 읽기
  - 파일 위치: output/chapters/[챕터명].md
  - 현재 줄수 확인
  - 기존 내용 파악

□ 주요 키워드 추출
  - 챕터 제목
  - 주요 섹션명
  - 핵심 개념
  - 고유 용어

□ 예상 PDF 범위 추정
  - 챕터 번호 기준 페이지 추정
  - 키워드 기반 검색 범위 설정
```

### 산출물
- 챕터 현황 메모
- 키워드 리스트
- 예상 PDF 파일 목록

## ✅ Phase 2: PDF 매핑 (각 챕터별)

### 목표
28개 PDF 중 해당 챕터의 원본 PDF 정확히 식별

### 체크리스트
```markdown
□ 28개 PDF 중 해당 챕터 PDF 식별
  □ 예상 PDF 파일 1개 선택
  □ PDF 읽기 (1개만!)
  □ 챕터 제목 검색
  □ 핵심 키워드 검색
  □ 페이지 번호 확인

□ 내용 일치도 확인
  □ 챕터 제목 일치 (90% 이상)
  □ 주요 섹션 일치
  □ 핵심 개념 포함 여부
  □ 예시/사례 일치 여부

□ 매핑 검증
  □ 올바른 PDF 찾았는지 확인
  □ 일치도 90% 이상인지 확인
  □ 다른 PDF 필요 시 재검색
```

### ⚠️ 주의사항
- **절대 금지**: 동시에 2개 이상 PDF 읽기 (API 413 에러)
- **순차 처리**: 한 번에 1개 PDF만 읽기
- **검증 필수**: 잘못된 PDF 선택 시 즉시 수정

### 산출물
- 챕터-PDF 매핑 정보
- PDF 내용 요약
- 일치도 점수

## ✅ Phase 3: 내용 분석 (각 챕터별)

### 목표
원본 PDF와 현재 마크다운 비교하여 누락 내용 식별

### 체크리스트
```markdown
□ PDF와 현재 마크다운 비교
  □ 전체 구조 비교
  □ 섹션별 내용 비교
  □ 예시 및 사례 비교

□ 누락된 내용 식별
  □ 누락된 섹션 목록 작성
  □ 누락된 스토리/사례 목록 작성
  □ 누락된 예시 목록 작성
  □ 누락된 설명 목록 작성
  □ 누락된 코드 예시 목록 작성

□ 추가할 내용 목록 작성
  □ 섹션별 우선순위 설정
  □ 예상 추가 줄수 계산
  □ 작성 순서 결정
```

### 산출물
- 누락 내용 목록
- 추가 내용 우선순위
- 예상 작업량 (줄수)

## ✅ Phase 4: 구조 설계 (각 챕터별)

### 목표
01장 구조를 기준으로 일관된 챕터 구조 설계

### 체크리스트
```markdown
□ 01장 구조 참고
  □ 01-observation.md 읽기
  □ 구조 패턴 분석
  □ 섹션 분량 파악

□ 섹션 구조 설계
  □ 들어가며 (100줄)
    - 챕터 소개
    - 핵심 개념 요약
    - 왜 중요한가

  □ 주요 섹션 4-6개 (600-700줄)
    - 섹션 제목 정의
    - 각 섹션 분량 배분
    - 순서 및 흐름 설계

  □ 핵심 정리 (50줄)
    - 주요 개념 요약
    - 공식 및 원칙
    - 좋은 예 vs 나쁜 예

  □ 체크리스트 (30줄)
    - 실행 가능한 항목
    - 검증 기준

  □ 마치며 (20줄)
    - 핵심 메시지 재강조
    - 다음 장으로 연결

□ 각 섹션별 내용 배분
  □ 섹션별 추가 내용 매핑
  □ 목표 줄수 설정
  □ 작성 순서 확정
```

### 산출물
- 챕터 구조 설계서
- 섹션별 내용 개요
- 목표 줄수 배분표

## ✅ Phase 5: 내용 작성 (각 챕터별)

### 목표
설계된 구조에 따라 상세 내용 작성

### 체크리스트

#### 5.1 들어가며 작성
```markdown
□ 챕터 소개
  □ 챕터 주제 명확히
  □ 전체 개요 설명

□ 핵심 개념 요약
  □ 주요 개념 3-5개 나열
  □ 간단한 설명 추가

□ 중요성 설명
  □ 왜 배워야 하는가
  □ 실무 적용 가치
  □ 다른 개념과의 연관성
```

#### 5.2 주요 섹션 작성 (4-6개)
```markdown
□ 섹션별 제목 명확화
  □ 구체적이고 명확한 제목
  □ 번호 체계 일관성

□ 스토리텔링 요소 추가
  □ 실제 상황 설정
  □ 등장인물 소개
  □ 문제 상황 묘사

□ 구체적인 예시 추가
  □ 코드 예시
  □ 실제 사례
  □ 수치 데이터

□ 대화 및 상황 묘사
  □ 대화체 사용
  □ 상황 상세 묘사
  □ 감정 및 반응 표현

□ 코드 예시 상세화
  □ 주석 추가
  □ 단계별 설명
  □ Before/After 비교

□ 단계별 설명 추가
  □ 프로세스 단계 명확히
  □ 각 단계별 설명
  □ 주의사항 및 팁
```

#### 5.3 핵심 정리 작성
```markdown
□ 주요 개념 요약
  □ 핵심 개념 정리
  □ 간결한 요약

□ 공식 및 원칙
  □ 핵심 공식
  □ 적용 원칙
  □ 예외 사항

□ 좋은 예 vs 나쁜 예
  □ 좋은 예시 제시
  □ 나쁜 예시 제시
  □ 비교 설명

□ 주의사항
  □ 흔한 실수
  □ 주의할 점
  □ 권장 사항
```

#### 5.4 체크리스트 작성
```markdown
□ 실행 가능한 항목
  □ 구체적인 액션 아이템
  □ 검증 가능한 항목
  □ 10-15개 정도

□ 검증 기준
  □ 완료 기준 명확히
  □ 측정 가능한 지표

□ 우선순위 표시
  □ 필수/선택 구분
  □ 순서 제안
```

#### 5.5 마치며 작성
```markdown
□ 핵심 메시지 재강조
  □ 가장 중요한 내용 1-2개
  □ 기억해야 할 것

□ 적용 방법 요약
  □ 실무 적용 방법
  □ 시작하는 방법

□ 다음 장으로 연결
  □ 다음 장 소개
  □ 연관성 설명
```

### 산출물
- 완성된 마크다운 초안 (800-1000줄)
- 섹션별 작성 완료 표시

## ✅ Phase 6: 품질 검증 (각 챕터별)

### 목표
작성된 내용의 정확성, 완전성, 구조, 품질 검증

### 체크리스트

#### 6.1 정확성 검증
```markdown
□ 원본 PDF와 100% 일치
  □ 내용 정확성
  □ 누락 없음
  □ 오타 없음

□ 기술 용어 정확성
  □ 전문 용어 정확
  □ 용어 일관성
  □ 번역 정확성

□ 수치 데이터 정확성
  □ 숫자 정확
  □ 단위 정확
  □ 계산 검증
```

#### 6.2 완전성 검증
```markdown
□ 누락 내용 없음
  □ Phase 3에서 식별한 모든 내용 포함
  □ 필수 섹션 모두 작성
  □ 예시 및 사례 충분

□ 모든 섹션 완성
  □ 들어가며 완성
  □ 주요 섹션 4-6개 완성
  □ 핵심 정리 완성
  □ 체크리스트 완성
  □ 마치며 완성

□ 체크리스트 완성
  □ 10-15개 항목
  □ 실행 가능한 내용
  □ 검증 기준 명확
```

#### 6.3 구조 검증
```markdown
□ 01장과 동일한 구조
  □ 5단계 구조 동일
  □ 섹션 순서 일관
  □ 분량 유사

□ 일관된 포맷
  □ 마크다운 문법 일관
  □ 제목 레벨 일관
  □ 코드 블록 형식 일관

□ 자연스러운 흐름
  □ 논리적 순서
  □ 섹션 간 연결
  □ 읽기 쉬운 구조
```

#### 6.4 품질 검증
```markdown
□ 목표 줄수 달성 (800-1000줄)
  □ 총 줄수 확인
  □ 섹션별 분량 확인
  □ ±10% 이내

□ 읽기 쉬운 문장
  □ 명확한 문장
  □ 적절한 길이
  □ 자연스러운 한국어

□ 실용적인 내용
  □ 실무 적용 가능
  □ 구체적인 예시
  □ 실행 가능한 조언

□ 명확한 예시
  □ 이해하기 쉬운 예시
  □ 충분한 설명
  □ 다양한 사례
```

### 산출물
- 품질 검증 보고서
- 수정 필요 항목 목록
- 최종 수정 완료 확인

## ✅ Phase 7: 최종 검토

### 목표
전체 챕터의 일관성 및 완성도 최종 점검

### 체크리스트
```markdown

  □ 다음 장으로의 연결
  □ 전체 흐름 자연스러움

□ 용어 일관성 확인
  □ 기술 용어 일관성

□ 네비게이션 링크 확인
  □ 이전 장 링크 작동

□ 마크다운 문법 확인

  □ 리스트 형식 올바름
  □ 테이블 형식 올바름

□ 파일 저장 및 백업

  □ 새 파일 저장
  □ 파일명 확인
  □ 저장 위치 확인
```
