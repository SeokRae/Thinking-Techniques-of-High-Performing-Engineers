# 2장: Method R 방법론

## Method R이란?

저자 캐리 밀샙이 개발한 **성능 최적화의 5단계 체계적 방법론**입니다.

핵심: 추측하지 말고, 측정하고, 데이터로 결정하세요.

---

## Method R의 5단계

### Step 1: 응답 시간 정의

**"무엇이 느린가?"를 명확히 하기**

```
예시: 청구서 생성
- 시작: "생성" 버튼 클릭
- 종료: "완료" 메시지 표시
- 현재: 330초
- 목표: 60초
```

### Step 2: 활동 측정

**"시간이 어디에 소비되는가?"를 파악**

```
청구서 생성 330초 분석:

데이터 조회:    100초 (30%)
계산:           180초 (55%) ← 병목!
문서 생성:       40초 (12%)
저장:            10초 (3%)
```

### Step 3: 영향 분석

**"어디를 개선할 것인가?"를 결정**

```
우선순위:
1위: 계산 (180초 - 가장 크다)
2위: 데이터 조회 (100초)
3위: 문서 생성 (40초)
4위: 저장 (10초)
```

### Step 4: 개선 실행

**"우선순위대로 개선한다"**

```
1단계: 계산 최적화 (180초 → 30초)
2단계: 데이터 조회 최적화 (100초 → 20초)
3단계: 캐싱 추가 (나머지 병목 제거)

최종: 330초 → 50초 ✓
```

### Step 5: 검증

**"실제로 개선되었는가?"를 확인**

```
개선 전후:
- 평균: 330초 → 50초 (85% 개선)
- p95: 400초 → 80초 (80% 개선)

결론: 목표 달성!
```

---

## 왜 이 방법이 작동하는가?

### 이유 1: 데이터 기반
```
추측: "CPU가 문제일 거야"
데이터: "계산이 55% 차지" ← 신뢰 가능
```

### 이유 2: 우선순위 명확
```
가장 큰 것부터 개선
→ 효율 최대화
→ 시간 절약
```

### 이유 3: 커뮤니케이션 개선
```
"시스템이 느려" (불명확)
vs.
"청구서 생성이 330초, 목표 60초" (명확)
```

---

## 체크리스트

Method R을 적용할 때:

```
□ Step 1: 응답 시간 명확히 정의
□ Step 2: 프로파일러로 측정
□ Step 3: 병목 분석 및 우선순위 결정
□ Step 4: 우선순위대로 개선
□ Step 5: 목표 달성 확인
```
