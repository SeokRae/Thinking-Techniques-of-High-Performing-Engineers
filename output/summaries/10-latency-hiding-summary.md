# 10장: 대기 시간 숨기기(Latency Hiding) - 요약

## 📌 핵심 개념

- **대기 시간은 없앨 수 없지만 숨길 수는 있다**
- **핵심**: "기다리는 동안 할 수 있는 일"을 찾기
- **작업 순서 재구성**: 대기를 겹쳐서 총 응답 시간 단축
- **병렬화**: 응답 시간은 줄이지만 총 작업량과 리소스는 그대로
- **사람 vs 시스템**: 시스템은 멀티태스킹, 사람은 집중력 집중

## 🎓 주요 교훈

**엄마의 초단시간 요리**
- 끓는 물 올리기 → (대기 시간에) 감자/샐러드 준비 → 재료 삶기
- 대기를 다른 준비 작업 뒤에 숨김
- **요리 총 소요 시간 단축**

**일상 속 적용**
- 골프: 샷 대기 중 클럽 준비·연습 스윙
- 광고: 화장실 다녀오기
- 주유: 앞유리 닦기
- 비행기: 탑승 대기 중 영화 다운로드, 메일 처리

**오라클 병렬화 데모**
- 2분 35초 쿼리 → 2초로 단축
- 병렬화 = 응답 시간 줄이기의 극단 형태
- **총 작업량은 그대로, 리소스 사용량 증가**

**선점형 멀티태스킹과 사람**
- 컴퓨터: 선점형 멀티태스킹으로 "동시에" 처리되는 것처럼 표현
- 사람: 컨텍스트 전환 비용이 매우 큼 (분 단위)
- **사람에게 멀티태스킹 강요하지 말기**

## 💡 실행 가능한 가이드

### 작업 순서 재구성
1. 대기 시간이 큰 작업을 먼저 시작
2. 그 사이 관련 작업(준비, 전처리) 수행
3. 결과 대기 중 다른 작업 병행

### 배치 작업 최적화
- 야간 백업 실행
- 비혼잡 시간대에 무거운 작업 예약

### 병렬화 활용
- 적정 병렬도 설정 (리소스 고려)
- 대기 시간이 병렬로 겹쳐질 수 있는 경우만

### 조직 구조
- 개발/운영 과정에서 사람 멀티태스킹 최소화
- 자동화와 시스템에 맡기기

## 📋 체크리스트

- [ ] 대기 시간이 긴 작업을 먼저 시작했는가?
- [ ] 그 사이 준비/다른 작업을 병행하는가?
- [ ] 배치 작업을 비혼잡 시간에 예약했는가?
- [ ] 병렬도를 현실적으로 설정했는가?
- [ ] 사람 멀티태스킹을 강요하지 않았는가?

## 🎯 한 문장 요약

> **대기 시간을 없앨 수 없다면 겹쳐서 숨기되, 사람은 집중력 집중, 시스템은 병렬화.**
