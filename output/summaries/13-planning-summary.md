# 13장: 계획(Planning) - 요약

## 📌 핵심 개념

- **용량 계획**: 자원 단위를 다른 단위로 바꿔 덧셈·곱셈하는 문제
- **모델 구성**: 실행/시간 × CPU초/실행 = CPU초/시간
- **최적화의 핵심**: 카운트 감소 또는 평균 시간 감소 (이 둘뿐)
- **현실적 기획**: 목표 수요, 여유 용량, 비용 구조를 맞추기
- **스프레드시트 활용**: 목표 탐색으로 파라미터 조정

## 🎓 주요 교훈

**용량 계획 모델 (월요일 14시 피크)**
- 기능별 실행/시간(B열): 하루 몇 번 실행?
- CPU초/실행(C열): 추적으로 얻은 단가
- CPU초/시간 = B × C
- 총 용량: 4코어 × 3,600초 = 14,400 CPU초/시간
- 남은 용량 = 총 - 소비, % 표시

**파라미터 조정의 효과**
- 실행 빈도 감소 (B열) → 전체 사용자에게 이득
- 단가 감소 (C열) → 전체 사용자에게 이득
- **최적화는 이 둘 중 하나뿐**

**현실적 기획**
- 과도한 목표/비현실적 용량 = 낭비 초래
- 원하는 결과(가용성/성능/비용)를 수량으로 표현
- 모델로 검증 후 의사결정

## 💡 실행 가능한 가이드

### 용량 계획 스프레드시트 구성

```
기능 | 실행/시간(B) | CPU초/실행(C) | CPU초/시간(B×C) | 여유(%)
-----+-------------|--------------|----------------+------
주문 |     500     |     0.1      |      50        | ...
급여 |      10     |     5.0      |      50        | ...
...
총합 |      -      |      -       |  1,234 / 14,400| 91.4%
```

### 의사결정 프로세스

1. **목표 수요 결정**: 피크 시간에 얼마나 처리할지
2. **여유 용량 설정**: 안정성을 위한 여유 % (예: 20%)
3. **필요 용량 계산**: 목표 ÷ (1 - 여유%)
4. **비용 구조 확인**: 필요 하드웨어 수
5. **합의**: 목표/비용/여유의 균형

### 파라미터 튜닝

- 엑셀 목표 탐색으로 "만약 실행 빈도를 50% 줄인다면?" 계산
- 행 하나 추가/제거 시 전체 영향 파악
