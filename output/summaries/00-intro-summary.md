# 0장: 서문 - 요약

## 📌 핵심 개념

- **최적화의 본질**: "무언가를 빠르게 만든다" = 그것을 "더 나은 것으로 만드는 것"
- **성능도 기능**: 측정·비교·관측 가능해야 하는 기본 요구사항
- **올바른 질문 먼저**: 기술적 답을 구하기 전에 "무엇이 정말 중요한가?"를 파악
- **측정이 핵심**: 추측이 아닌 데이터 기반 의사결정
- **정치·심리 이해**: 기술적 해결책과 함께 조직·감정 요소까지 다루기

## 🎓 주요 교훈 (사례 기반)

**저자의 25년 경험**
- 1990년대 오라클 컨설턴트로 수백 고객사 지원
- 『Optimizing Oracle Performance』 공저
- Method R Corporation 설립 (2008), 성능 추적 분석 도구 개발
- **교훈**: 현장 기반의 검증된 경험과 방법론이 조직을 뛰어넘어 적용 가능

## 💡 실행 가능한 가이드

1. **순서대로 읽기**: 개념 → 방법론 → 측정 → 최적화 → 사례로 단계적 학습
2. **필요시 참조식 읽기**: 특정 주제(지연, 낭비, 예측 등)만 찾아 읽기 가능
3. **실무 적용**: 방법론과 체크리스트로 바로 프로젝트에 활용
4. **재검토**: 언제든 다시 꺼내 최신 프로젝트에 적용

## 📋 체크리스트

- [ ] "무엇이 정말 중요한가?"를 먼저 물어봤는가?
- [ ] 추측이 아닌 측정 가능한 데이터를 준비했는가?
- [ ] 기술적 해결책과 함께 조직 정치를 고려했는가?
- [ ] 성능을 기능처럼 설계/측정 가능하게 했는가?
- [ ] 팀원들과 문제 정의를 공유했는가?

## 🎯 한 문장 요약

> **성능 최적화는 기술과 정치가 만나는 지점에서, 올바른 질문과 측정을 통해 시작된다.**
