# 11장: 논리적 오류(Logical Fallacies) - 요약

## 📌 핵심 개념

- **잘못된 대리자(Surrogacy)**: CPU 사용률 낮음 = 사용자 만족? 아님!
- **원하는 것 직접 측정**: 대리 지표보다 사용자 경험
- **비율의 함정**: 평균과 맥락 없이는 틀린 결정
- **이상치 주의**: 평균 뒤에 숨은 분포와 극값
- **명확한 설명**: 절대값 함께 제시 (배수나 %만은 위험)

## 🎓 주요 교훈

**사악한 램프 요정 지니**
- CPU 사용률이 낮으니 문제 없다? → 사용자는 여전히 느낄 수 있다
- **원하는 것(사용자 경험)을 직접 측정해야 함**

**가죽 자켓의 비율 함정**
- 자켓 평균 가격을 낮추려고 또 사는 건 숫자 놀음일 뿐
- **비율/평균만으로는 올바른 결정 불가능**

**묻혀 있는 이상치**
- 100만 번 중 1번의 18.6초 호출이 평균에 0.001초로 묻힘
- **분포를 보고 이상치를 무시하지 말기**

**잘못된 목표**
- IT 본부장: "CPU 대기율 40%"를 원한다
- 실제: 사용자 경험 개선
- **정말 원하는 것을 확인하기**

## 💡 실행 가능한 가이드

### 올바른 지표 선택

1. **직접 측정**: 대리 지표보다 목표에 직결된 지표
   - CPU 사용률 × → 응답 시간 ○

2. **비율 해석 시 맥락 포함**
   - 캐시 적중률 단독 × → 절대값 함께 ○
   - "캐시 적중률 99% (1000개 요청 중 1개 미스)" ○

3. **절대값과 함께 표현**
   - "1,000% 개선" × → "10초→1초 (90% 감소, 10배)" ○

4. **분포 확인**
   - 평균 시간 ×
   - 분포, 최소/최대, 이상치 ○

### 개선 설명법

- **p = (old - new) / old** (개선율, %)
- **n = old / new** (배수)
- **반드시 old/new 값을 함께** 제시

## 📋 체크리스트

- [ ] 직접 측정할 수 있는 지표를 선택했는가?
- [ ] 대리 지표만 보지 않았는가?
- [ ] 비율을 제시할 때 절대값을 함께 했는가?
- [ ] 평균뿐 아니라 분포를 확인했는가?
- [ ] 이상치를 무시하지 않았는가?
- [ ] 정말 원하는 것이 무엇인지 확인했는가?

## 🎯 한 문장 요약

> **지표와 비율은 쉽게 속이므로 절대값, 분포, 맥락을 함께 봐야 한다.**
